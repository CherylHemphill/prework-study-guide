HTML
  • HTML is the frame for the site.
  • We use semantic HTML elements to indicate the role of their content, as well as to give structure to the content of the webpage.
  • The head element contains information about the webpage.
  • The body element represents the visible content shown to the user.
  • Declarations are not case sensitive but elemenst are case sensitve.
  • The head contains metadata and metadata elements contain information about the page.
  • The br element creates a line of empty space.
  • The a element stands for anchor and creates links to the same webpage or other webpages.
  • The href attribute points to the URL for the the link
  • The element div creates a divison is the page and element span creates an inline division.
  • When adding an imagine the src attribute defines the location, or source, of the image file. The alt attribute contains a string of text that describes the appearance and function of the imagine.
  • HTML uses the id attribute as a way to identify elements and select them.
  • Closing tags begin with the forward slash
  • When creating bullet pouint list the element ul creates the unordered list. Following lines with the element li create the line for information input.

Git
  • git status : checks what branch we are currently on
  • git checkout -b branch-name :creates a new branch and switches to it
  • git add -A : add command adds modifications in the current working branch and the -A flaf indicates that we want to add all changes
  • git pull origin main : pull command to receive a branch's modifications into the local enviroment. Origin keyword indicates the source of the pull will be in the GitHub repo from the main branch.
  • It is important to always do a git pull to make sure that you have the latest version of the project on your local computer.
  • git push origin feature/branch-name : used to push the changes we have locally to our remote GitHub branch
