HTML
  • HTML is the frame for the site.
  • We use semantic HTML elements to indicate the role of their content, as well as to give structure to the content of the webpage.
  • The head element contains information about the webpage.
  • The body element represents the visible content shown to the user.
  • Declarations are not case sensitive but elemenst are case sensitve.
  • The head contains metadata and metadata elements contain information about the page.
  • The br element creates a line of empty space.
  • The a element stands for anchor and creates links to the same webpage or other webpages.
  • The href attribute points to the URL for the the link
  • The element div creates a divison is the page and element span creates an inline division.
  • When adding an imagine the src attribute defines the location, or source, of the image file. The alt attribute contains a string of text that describes the appearance and function of the imagine.
  • HTML uses the id attribute as a way to identify elements and select them.
  • Closing tags begin with the forward slash
  • When creating bullet pouint list the element ul creates the unordered list. Following lines with the element li create the line for information input.
CSS
  • CSS is a programming language used by developers to define how webpage content is presented to users.
  • There are three ways to style a webpage using CSS: inline CSS, internal CSS style sheet, and external CSS style sheet.
  • Inline styles apply to a specific HTML tag.
  • Internal style sheet contains CSS rules for the webpage in the head section of the HTML file.
  • External CSS style sheet is preferred in most cases because it allows developers to keep all their CSS rules in a separate file.
  • When using external CSS a link element is used in the head to place the style in the HTML.
  • The rel attribute specifies the relationship between the current document and the linked document resource.
  • The href attribute specifies the location (URL) of the external resource.
  • The wild card symbol, *, is used to apply a rule to all the elements visible on the page.
  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.
Git
  • git status : checks what branch we are currently on
  • git checkout -b branch-name :creates a new branch and switches to it
  • git add -A : add command adds modifications in the current working branch and the -A flaf indicates that we want to add all changes
  • git pull origin main : pull command to receive a branch's modifications into the local enviroment. Origin keyword indicates the source of the pull will be in the GitHub repo from the main branch.
  • It is important to always do a git pull to make sure that you have the latest version of the project on your local computer.
  • git push origin feature/branch-name : used to push the changes we have locally to our remote GitHub branch
  • git commit -m "message" : the -m flag associates a message with the commit. Commit messages should be short descriptive messages of the changes made.
  • It is a good habit to always pull before we push to make sure it is up to date.
  • After updating code is VS return to command line and prepare for merge with git commands : git add -A , git commit -m "message" , git pull origin main , git push origin feature/nameofbranch
  • After merging in GitHub return to terminal and begin pull to local computer: git checkout main , git pull origin main
